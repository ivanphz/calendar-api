# 事件模型契约 —— EVENT-MODEL (v6 宪法)

> **这份文档是什么**:域 ↔ 框架之间【日历通道】的数据契约。v6 起,域不再产出 ICS 文本,
> 只产出本文档定义的**事件对象**;一切 ICS 语法(转义/折行/DTSTART/VALARM/时区换算)
> 由框架渲染器独家负责。
>
> **地位**:本文档是契约的唯一真相源。ARCHITECTURE §3、DEVGUIDE 接入手册与本文冲突时,
> 以本文为准(它们随后修订对齐)。实现先于本文档定稿 = 违规。
>
> **不覆盖什么**:闹钟通道(?format=json)协议 v1 **原样不动**,权威见 DOWNSTREAM.md;
> 本文只在 §4 交代两套 uid 协议的分界,防止混用。

---

## 0. 总纲(一句话 + 一条石蕊测试)

> **框架吃语法,域吃语义。**

- 域交出**"发生了什么"**:哪一刻(墙上时间+时区)、什么标题、什么正文、想在什么时候被提醒。
  全部是**原始语义值**——真实换行、真实逗号、IANA 时区名、纯数字分钟数。
- 框架负责**"怎么写成 ICS"**:转义、折行、CRLF、日期时间语法、VALARM 语法、VCALENDAR 信封。

**石蕊测试**(裁决任何一行代码归谁):
*"如果哪天输出格式从 ICS 换成 Google Calendar API,这行代码要不要动?"*
要动 = 框架;不动 = 域。

**红线**:框架永远不做业务合并。"同日 3 笔账单合成一个事件"是域的业务决策
(牵扯 tags 过滤先于合并、标题折叠规则)。域先合并、再 emit 一个事件对象;
框架看到的是"最终要出现在日历上的事件清单",一对一渲染,不增不减不并。

## 1. 事件对象(Event)

```js
{
  // ── 身份块 ──
  uid: 'card-day-20260715',            // 必填。规则见 §4(日历侧协议)

  // ── 时间块(二选一形态) ──
  // 形态 A:全天事件
  allDay: true,
  date: '2026-07-15',                  // 必填。事件日(本地历法日,无时区概念)
  endDate: '2026-07-16',               // 可选。多日事件的结束日(【排他】,ICS 惯例:
                                       //   不填 = date 次日,即单日全天事件)

  // 形态 B:定点事件
  allDay: false,
  date: '2026-07-15',                  // 必填。事件日(tz 时区下的墙上日期)
  time: '09:30',                       // 必填。墙上时刻 HH:MM(24h)
  tz: 'Asia/Shanghai',                 // 可选。IANA 名;缺省 = HUB_CONFIG.timezone
  durationMinutes: 360,                // 必填(定点事件)。显式优于默认,不设框架缺省

  // ── 内容块(全部原始文本,见 §2 铁律) ──
  summary: '💳Repay 2笔: CMB + 🇭🇰HSBC-HK', // 必填
  description: '【合并提醒】…\n\n────\n…',   // 可选。真实换行,严禁手写转义
  url: 'https://…',                    // 可选。渲染为 URL 属性 + 由域自行决定是否写进正文

  // ── 提醒块(VALARM 意图,见 §3) ──
  reminders: [ … ],                    // 可选。缺省 = 无提醒(事件照常进日历)

  // ── 归属块 ──
  tags: ['life'],                      // 可选。目前仅进诊断统计;为框架级过滤预留
  ext: { }                             // 可选。自由袋,框架不读不校验(前向兼容约定,
                                       //   与 config/checkin.js 的 ext 同一哲学)
}
```

### 字段裁决理由(节选)

- **date+time+tz 而非 UTC 瞬间**:两者数学等价(本系统全是展开好的具体实例,无 RRULE),
  但墙上时间是**业务原话**("东京 9 点开盘"),UTC 是**推导结论**。域说原话、框架做推导
  —— facts not conclusions。且诊断报告可原样打印 `Asia/Tokyo 09:00`,排错不用心算。
- **durationMinutes 而非 end 时刻**(定点事件):跨日、DST 边界下 duration 无歧义。
- **定点事件 duration 必填,无框架缺省**:explicit over defaults。全天多日才用 endDate。
- **endDate 排他**:与 ICS DTEND;VALUE=DATE 语义一致,消灭 ±1 天换算错位。

## 2. 时区与文本两条铁律

### 铁律一:域绝不自己换算时区

域只声明"当地几点"(墙上时间 + IANA 名),换算是消费方(框架渲染器)的事。
与闹钟通道"异时区业务别自己换算,带 tz 字段交给网关"(DOWNSTREAM §2)**完全同构**
—— 三层生态里,时区换算永远发生在消费侧,全链只此一个方向。

- `tz` 只收 **IANA 名**(`Asia/Tokyo`、`America/New_York`、`Etc/UTC`)。
  不收 `+09:00` 偏移量:TZID 需要名字,且裸偏移无 DST 信息,是伪时区。
  只有偏移没有城市的业务源,域负责映射到一个 IANA 名(拿不准就 `Etc/UTC` + 换算好)。
- 缺省 = `HUB_CONFIG.timezone`(东八)。现有两个域零负担,一个 tz 都不用写。

### 铁律二:内容块只收原始文本

summary/description/url 里:真实换行就是 `\n`(JS 源码单反斜杠),逗号就是逗号。
**严禁**出现手写的 ICS 转义序列(`\\n`、`\\,` 等源码双反斜杠字面量)。
转义、UTF-8 安全折行(75 字节,不劈多字节字符)全归渲染器。

**探雷断言(迁移期保险丝,长期保留)**:渲染器检测到原始文本中含
字面 `\n` 序列(反斜杠字符 + n)即在诊断响亮告警 —— v6 之后域没有任何正当理由
手写转义序列,出现即迁移漏网或新插件误学旧写法。

> 迁移注意:v5 源码里的 `\\n` 是"源码层反斜杠+n → 落进 ICS 成为 RFC 换行转义"。
> 迁移时须全部降级为真 `\n`。漏改 = 正文出现字面 \n;多改 = 双重转义。
> 两个方向都静默,靠上述断言 + 金标准 diff 双保险。

## 3. 提醒块(VALARM 意图)

两种意图形态,对应现实中的两种业务(card 的 exact/allday 双模式都装得下):

```js
// 形态 1:相对提前(用于定点事件)
{ minutesBefore: 5 }                       // 0 = 准点

// 形态 2:锚定日的绝对钟点(用于全天事件;即原 allDayReminders 三元组)
{ dayOffset: -1, at: '20:00' }             // 事件日前一天 20:00
{ dayOffset: 0,  at: '09:30' }             // 事件日当天 09:30

// 两种形态均可选带:
{ …, label: '还款日前一晚提醒!' }           // 提醒弹窗文案;缺省用框架默认文案
```

- 形态与事件类型**建议配对**(定点→形态1,全天→形态2),渲染器都能翻译,不强校验。
- 渲染器**只产出相对 TRIGGER**(`-PT…M` / 带符号 duration):形态 2 由渲染器换算成
  相对量再输出。**接口上不提供绝对 TRIGGER** —— 这是把"Apple 把无限定绝对
  DATE-TIME TRIGGER 当 UTC 读、偏 8 小时"的历史血债做成**结构上不可能再犯**。
- label 归域(它是文案不是语法),但缺省文案归框架(两个现存域都用缺省)。
- reminders 只管日历 VALARM。闹钟网关条目在 alarms 通道,两者**完全无关**
  (延续 config/card.js "命名刻意不含 alarm"的既有分界)。
- iOS 已知行为(写给未来插件作者,省一次真机排错):订阅日历的整册
  "Event Alerts"开关盖过一切 VALARM —— 提醒不响先查手机侧总开关,不是你的 bug。

## 4. 两套 uid 协议(分界,防混用)

| | 日历事件 uid(本契约) | 闹钟 uid(协议 v1,DOWNSTREAM) |
|---|---|---|
| 消费方 | iOS 日历(整册替换式刷新) | alarm-api 网关(增量对账) |
| 时钟时间 | **允许**(身份可以就是"这一天":`card-day-20260715`) | **严禁**(bucket 锚事件不锚响铃,否则改时间=僵尸闹钟) |
| 字符集 | ASCII 可打印;建议 `[A-Za-z0-9_.-]` | 硬限 `[A-Za-z0-9_.-]` |
| 长度 | 无下游硬限;建议 ≤60 | **硬限 ≤40**(网关拼标签) |
| 前缀 | **必须 = 域 id + `-`**(跨域天然不撞,框架校验) | 同左 |
| 稳定性 | 建议稳定(刷新间 uid 漂移 = iOS 视为删旧建新,无功能损害但脏) | 必须稳定(网关平滑不抖的前提) |
| @后缀 | 渲染器统一追加 `@mycal.local`,域**不要**自带 | 无 |

框架出口校验(两通道都查,实现在 src/governance.js),**软硬分层**:

- **闹钟侧全硬**(字符集/长度≤40/前缀,违规=熔断该条):这是下游网关拼标签的物理现实,
  违规条目流过去只会被下游静默计入"无uid",排错要跑到下游面板 —— 必须在本框架显形。
- **日历侧**:前缀与全 feed 唯一性**硬**;字符集/长度**响亮告警不熔断** ——
  iOS 实测容忍 Unicode uid(v5 现网 PAB '一账通' 的日历 uid 即含中文),硬杀会误伤存量;
  告警持续挂诊断催清理,card 手术 B 换短哈希后自然消音。
- 任何情况下**绝不静默修复** —— 静默修复会让同一 uid 在两次请求间被"修"得不一样,
  反而制造漂移;配置错误必须显形(与 coverage 告警、cnRule 告警同一哲学:响亮降级)。

## 5. 适配器契约 v2

```js
export const xxxDomain = {
  id: 'xxx',
  contract: 2,              // v6 契约标记
  calName: '…',
  defaultColor: '#RRGGBB',
  window: 'USE',            // 治理姿态:USE | MAP | OWN(见 §6)

  prepare(q, ctx) {         // 可为 async,框架一律 await
    return { countries: ['CN'], state: {/* 域内私有 */} };
  },

  async build(state, hub, ctx) {
    return {
      events:     [ /* §1 事件对象 */ ],
      alarms:     [ /* 协议 v1 条目,原样不变 */ ],
      debugLines: [ /* 诊断段落,字符串,现状沿用 */ ]
    };
  }
};
```

**【ctx 与位置参数的分工规则】**(记住这一条就不会问"XX 该放哪"):

> **ctx = 请求级【恒定】的一切**(从头到尾不变)。
> **hub = 【阶段二才诞生】的资源** —— 它要等 prepare 报完 countries 才能创建,
> 进不了 prepare 的 ctx,故走位置参数。

**ctx(框架下发的上下文,替代旧的零散参数)**:

| 字段 | 说明 |
|---|---|
| `env` | KV / Secret。**prepare 也拿得到** —— 账户列表存 KV 的域,得先读 KV 才知道自己要哪些国家(v5 的 `prepare` 第4参能力,原样保留) |
| `baseDate` | 基准日 Date(含 testDate 覆盖,即旧 baseDateObj) |
| `todayStr` | 'YYYY-MM-DD'(hub 时区) |
| `window` | `{ from: 'YYYY-MM-DD', to: 'YYYY-MM-DD' }` **绝对日期区间,闭区间**。框架由统一参数换算;域按需 MAP 成自己的语义(如账单月循环边界) |
| `filters` | `{ tags, excludeTags }`(现状) |
| `matchesTags(item)` | 框架提供的标签命中助手 —— 消灭两个 adapter 里逐字重复的 hitTags |
| `nowBeijingStr` | 人读运行时间(旧 beijingTimeStr,诊断用) |
| `hubTimezone` | HUB_CONFIG.timezone,域给事件缺省 tz 时的参照(通常不用管) |

**对 v5 的三处收敛**:

1. `prepare` 不再返回 `years` —— 窗口归框架后,"需要哪几年假期"由框架从
   `window ± 1 个月缓冲`直接推导。两个域里两套算法不一致的年份推导代码删除。
   (缓冲覆盖:提前量把提醒推到窗口边界外一点、假期顺延跨年边界。)
2. `build` 第 ①项从 `eventLines: string[]` 变为 `events: [Event]`。
   card adapter 的 `indexOf('BEGIN:VEVENT')` 切片 hack 随之消亡。
3. dtStamp / beijingTimeStr 位置参数取消 —— DTSTAMP 是 ICS 语法归渲染器;
   运行时间进 `ctx.nowBeijingStr`。
4. `build` 的 `env` 位置参数取消 —— env 是请求级恒定资源,归 `ctx.env`,
   全链**只此一处**(不留"两种拿法"的歧义)。

## 6. 窗口治理(总控参数落点)

- 中枢统一参数:`?past=` `?future=`(单位:月;沿用 card 现有参数名**升格为中枢级**,
  旧链接语义不变)。缺省值住 `config/hub.js`。checkin 的 `?months=` **废除**
  (v6 破坏性变更,迁移时同批盘点存量链接,见 MIGRATION 计划)。
- 框架将其换算为**绝对日期区间**下发(月对不同域语义不同 —— card 是账单月、
  checkin 是 30 天节律,月是个骗人的单位;绝对日期无歧义)。`testDate` 天然是基准日。
- **窗口是【许可边界】,不是【生产配额】**:域只被约束"不得越界",从不被要求"填满"。
  签到域算法天生只向前推,past 区间自然空产出 —— 空即空,框架不催产;
  信用卡域回溯有真实价值(核对推算、暴露 bug),它用足这个许可。
  同一个 past 参数,两种用法都合法。
- **两条腿,缺一不可**:
  - **下发**(协商):窗口进 ctx,域自愿用来省算力(card 少循环、email 域少读 KV)。
  - **兜底**(强制):build 之后框架按事件 `date`(事件自己的本地墙上日期)硬裁,
    裁剪必上诊断:`⚠️ 域 card 越界输出 4 条已裁剪(裁剪窗 … ,含 ±45 天缓冲)`。
  只下发是君子协定,只兜底是白算一遍;信任但验证。
- **裁剪窗 = 视图窗 ± 45 天结构性缓冲**(常量,实现在 governance.js 并附论证):
  裁的是"离谱越界",不是"贴边溢出"。宽限跨月(cycle 模型 statementDay 晚 + 21 天)、
  假期顺延、提前量把提醒推出边界 —— 这些是**业务**不是越界,归域的 MAP 语义,
  框架不掺和。假期 years 推导同样基于裁剪窗,天然覆盖全部合法溢出。
- **治理姿态声明**(adapter 静态属性,不进 URL —— URL 是"这次怎么看",
  adapter 是"这个域天生什么脾气"):
  - `USE` 直接吃框架窗口;`MAP` 映射成自己的语义(card→账单月);
  - `OWN` 完全自持,**必上诊断显形**(否则是暗箱)。v6 两个现存域为 USE/MAP,
    OWN 是给未来"窗口概念不适用"的插件留的合法出口,规范在此、暂无使用者。
- **豁免边界(硬规矩)**:只影响自己的可豁免(窗口/历史保留);影响别人的不可豁免
  (uid 合法性、超时、输出体积)—— OWN 也逃不掉出口校验,fail-closed。

## 7. 渲染器保证(域可以依赖的事实)

渲染器是**所有 iOS 血债的唯一收容所**。域侧写作零 ICS 知识,自动继承以下全部:

| 保证 | 细节 |
|---|---|
| TEXT 转义 | `\` `;` `,` 换行,RFC 5545;域给原文即可 |
| 折行 | 75 字节,UTF-8 安全(不劈多字节字符) |
| CRLF | 全文档统一 |
| 定点时间 | `DTSTART;TZID=<IANA>:<墙上时间>` + `DURATION:PT…M`(继承 v5 pushTiming 语义) |
| 全天时间 | `DTSTART;VALUE=DATE` + 排他 DTEND |
| VALARM | 只产相对 TRIGGER;绝对 TRIGGER 在接口上不存在(§3) |
| DTSTAMP/@域名后缀 | 统一生成/追加 |
| 信封 | VCALENDAR 头、册名/颜色、诊断事件、哨兵事件(现状语义原样) |
| 排序 | 事件按 date 升序落盘(iOS 不在乎,人排错在乎) |

**已记录的依赖(决策记录,非疏漏)**:TZID 引用 Olson 名而**不携带 VTIMEZONE 块**
—— RFC 严格说要求,Apple 认裸 TZID。哪天要喂非 Apple 客户端,补 VTIMEZONE 只改
渲染器一处,这正是本次重构买来的权利。此依赖写进渲染器头注。

## 9. 裁决记录(v6 定稿)

| # | 裁决 | 理由摘要 |
|---|---|---|
| 1 | 账户身份 = **crc32 短哈希**(~~手写短名 alias~~,已推翻) | 见下方【裁决 1 的翻案记录】 |
| 2 | email 模型 KV 键**直接换哈希,不留双键** | Ivan 确认 email 模型未启用、KV 无真实数据 —— 零成本改键的最后窗口,用掉它 |
| 3 | 中枢窗口缺省 `past:3, future:12`(住 config/hub.js);**checkin 姿态 = MAP(只吃 future,忽略 past)** | 迁移期金标准要求行为全等,签到册凭空多 3 个月历史事件不算全等。以后想要历史签到,姿态改 USE 即可,一字之改 |
| 4 | 提醒缺省文案 = **v5 原文逐字**("提前N分钟提醒!"/"第X天 HH:MM 提醒!"/"事件开始时提醒!") | 金标准 diff 全等。迁移后想改随时改,一处生效 |

### 裁决 1 的翻案记录(值得留着 —— 它定义了 uid 这个概念本身)

**原主张:手写短名 `alias`。** 论据:①手机标签 `Gate-ES-<code>-<uid>-<HHMM>` 里的 uid 要人能看懂;
②显示名与身份解耦(改卡名 uid 不抖)。

**被推翻,理由成立:**

> **uid 的工作是【配对】,不是【沟通】。**

- 沟通归日历 `summary` 与闹钟 `reason` —— 那里全是人话;
- 手机标签里的 `code`(=`repay`)**已答完"哪个项目"**,剩下的"哪张卡"本就是日历的活;
- "改卡名会churn"的实际后果只是旧关新建、**一次刷新自愈**,是个眨眼不是灾难 —— 原主张把它说重了。

**可读性被放错了层。** 正确的补偿不是改 uid,而是:**治理诊断打 uid 时必带 summary/reason**
(实现在 governance.js 的 `who()`)。身份保持稳定 + 人保持看得懂,两件事本来不冲突 ——
是"改 uid"这个手段把它们绑在了一起。

**哈希白捡的**:零配置、Unicode 原生(根治 v5 有损 ASCII 压缩的中文撞键 bug)、不会手滑写重。

### 同批否决:框架侧"按长度自动转哈希 + URL 参数控制"

提案:框架在输出闹钟时,对超过 N 字符的 uid 自动转哈希,并用 URL 参数开关。**否决,三条**:

1. **长度阈值 = 悬崖**。CMB 那条 38 字符,改个卡名多 3 个字就越过 40 → uid 从明文**悄悄**翻成
   哈希 → 网关全体重建,而没有任何东西告诉你为什么。
   **身份不能取决于"这串字符碰巧多长"这种偶然属性。**
2. **URL 参数控制身份 = 身份跟着链接走**。你浏览器里自检看到明文 uid,网关拉的那条是哈希 uid
   —— **调试看的和生产跑的不是一个东西**。视图参数改的是"你看什么",身份参数改的是"东西是谁";
   两个物种,不能共用一套参数系统。
3. **框架不知道身份的结构**。域侧哈希能做到 `card-<账户哈希>-<账单月>`(月份还看得见);
   框架侧只能把整串搅进去变成 `card-<一坨>`。**只有域知道哪一段是账户、哪一段是月份** ——
   这与"框架不许合并事件"是同一条理。

**结论:哈不哈、哈哪段,发生在【设计时】(人看一眼自己的 uid,定一次,冻住),
不在【运行时】。** 设计时判断产出的是稳定常量;运行时判断产出的是会跳的变量。

### 三层分工(这才是可复用的结论)

| 层 | 归谁 | 落点 |
|---|---|---|
| **约束**(≤40 / 字符集 / 前缀=域id) | **框架** | `governance.js` 的 `checkAlarmUid` |
| **工具**(一个 crc32 函数) | **框架发,插件自愿用** | `governance.js` 的 `uidHash` —— 与校验器**一造一查、同一件事的正反面**,故住一起;第二个要哈希的域不必再抄一份 |
| **策略**(哈不哈 / 哈哪段) | **插件自己定** | 判据见 DEVGUIDE §5.2 |

**不存在"信用卡模式"这种东西**,只有一条判据:
*身份天生短且 ASCII → 直接用;身份长/含中文/机器生成 → 哈希。*
checkin 的 `checkin-moeshare-202607` 按此判据**不哈希**(强制全域哈希是纯亏)。同判据,不同答案。

**选了哈希 = 欠一条冻结声明**(哈希输入是哪几段、什么顺序、承诺不改),
并**用测试当执行器** —— 声明不靠自觉。
