# v5 → v6 升级 runbook(一次性)

> **这份文档的寿命**:v6 落地并观察稳定后即可删除,与 UPGRADE-V5.md 同待遇。
> 它记录的是**一次性动作**,不是长期规矩;长期规矩在 ARCHITECTURE / DEVGUIDE / EVENT-MODEL。
>
> **v6 是什么**:Tier 2 重构 —— 域不再产 ICS 文本,改交【事件对象】,框架独家渲染 + 统一治理。
> 决策全文见 DEVLOG v6.0 / v6.1。

## 0. 你要动手的只有一件事:盘点存量链接

代码侧全部改完并测绿了。**唯一需要人肉的是订阅链接**。

三处可能藏着旧链接:

1. **iPhone 日历订阅**(设置 → 日历 → 账户 → 已订阅的日历)
2. **alarm-api 网关的 SOURCES 配置**(拉 `?format=json` 的那条 URL)
3. **你自己的浏览器书签 / 备忘录**

对照下表逐条检查:

| 旧写法 | v6 行为 | 改成 |
|---|---|---|
| `?months=6` | 按 `?future=` 缺省(12)处理 + **诊断响亮告警** | `?future=6` |
| `?adAlarms=-1:20:00` | 按 `config/card.js` 的 `allDayReminders` 缺省处理 + **诊断告警** | `?allDayReminders=-1:20:00` |
| `?exAlarms=60` | 按 `exactReminders` 缺省处理 + **诊断告警** | `?exactReminders=60` |
| `?past=` / `?future=` | ⚠️ **语义扩大了**:v5 只管信用卡,v6 **统管全域**(签到也跟着收窄) | 想清楚是否要;不要就删掉用缺省 |
| `?cnRule=official` | 按 bank 处理 + 诊断告警(v5 就废了) | `?cnRule=bank` 或删掉 |

**链接光秃秃的(如 `?cal=checkin`、`?format=json`)= 什么都不用改**,推上去即可。

### 怎么确认改干净了

打开任意一条 ICS 链接(**不加 `?debug=0`**),看诊断事件正文里的 **【出口治理 · 框架】** 段:

```
【出口治理 · 框架】
  ✅ 无越界/无 uid 违规
```

以及顶部窗口行 —— 如果有废除参数,会有 `⚠️ ?xxx= 已废除/已删除 …… 请更新此链接`。

> **为什么废除的参数"不崩、只告警"**:旧链接会"看起来正常"却悄悄跑偏 ——
> `?months=6` 的人悄悄拿到 12 个月、`?adAlarms=` 的人悄悄拿到默认提醒。
> **不报错、只是行为悄悄变了,是最坏的那种失败。** 所以一律显形,你看得见才改得掉。

## 1. 破坏性变更全清单

### URL 参数

- **废除** `?months=`(签到域跨度)→ 并入中枢 `?future=`
- **删除** `?adAlarms=` / `?exAlarms=`(v5.2 起的兼容别名)→ `?allDayReminders=` / `?exactReminders=`
- **升格** `?past=` / `?future=`:card 私产 → **中枢参数**,缺省搬到 `config/hub.js`

### 配置(config/card.js —— 你的领地,已代你改好)

- **删除** `needsFxPurchase` / `fxNote`(7 处账户 + 2 处注释样板)—— 预留接口,从未启用。
  违反"explicit over speculative"的投机字段;真遇到购汇卡,事件对象的 `ext` 自由袋随时能装。
- **删除** `DEFAULT_CONFIG.pastMonths` / `futureMonths` —— 窗口归中枢后成死配置,
  缺省搬到 `config/hub.js`(值不变:3 / 12)。

### 身份(uid / KV 键)

- card **日历 uid 前缀** `repay-` → `card-`(前缀=域id 是治理的跨域防撞硬规则)
- card **账户身份**:五段长主键 → **crc32 短哈希**(`accountKey`)
  - 日历 uid:`card-<哈希>-<账单月>-<提醒日>`
  - 闹钟 uid(each):`card-<哈希>-<账单月>`(20 字符;v5 汇丰那条 44 > 40)
  - KV 键:`stmt:<哈希>` / `hist:<哈希>:<ISO>`

**这些变更对你意味着什么:**

| 通道 | 影响 | 为什么安全 |
|---|---|---|
| iOS 订阅日历 | 全部 uid 换新 | 订阅日历是**整册替换式刷新**,iOS 无感 |
| 闹钟网关 | 全部 uid 换新 | 你自己的对账规则:"不再吐该 uid = 自动关",旧的关、新的建,**一次刷新自愈** |
| KV(email 模型) | 键全换 | email 模型**未启用、KV 无真实数据** —— 零成本改键的最后窗口,用掉了 |

⚠️ 唯一注意:**切换当天别让新旧两版在线上交替服务**(比如回滚又前滚),
否则网关看到 uid 反复出现消失会空转建删。一次推上去、绿了就别回头。

### 内部契约(与你无关,除非你写插件)

- `prepare(q, ctx)` / `build(state, hub, ctx)`;域不再自报 `years`;
  `eventLines: string[]` → `events: [事件对象]`
- 详见 **docs/EVENT-MODEL.md**(宪法)与 **docs/DEVGUIDE.md**

## 2. 修掉的真 bug(顺带的)

| bug | v5 症状 | v6 |
|---|---|---|
| **闹钟 uid 超长** | 汇丰卡 44 字符 > 下游硬限 40。`cardAlarm=each` 下 **13 条闹钟被网关静默丢弃**,只在下游面板记"无uid" | 20 字符,全数存活;超限现在会在**你的日历诊断**里熔断报数,不必跑下游面板 |
| **中文卡名撞键** | `asciiId()` 有损压缩把中文抹掉 → 两张中文简称的 CNY 无尾号卡压成同一个 id → **闹钟互相顶掉**。而 config 注释同时允许"简称可中可英"+"合并账单 last4 留空",两条规则一叠加就踩雷 | crc32 原生吃 Unicode,根治 |
| **ICS 零转义** | 域是裸字符串拼接,卡名里写个逗号(`'Pulse, Red'`)就**破行** | 渲染器统一转义,结构性消失 |
| **ICS 零折行** | 长正文单行几百字节(Apple 容忍,严格 parser 未必) | 75 字节 UTF-8 安全折行 |
| **年份推导两套** | 两个域各写一套算法**还不一致**(card 有 ±1 冗余,checkin 用 Math.ceil) | 归中枢,一处 |
| **hitTags 重复** | 两个 adapter 各抄了一份逐字相同的实现 | 收归 `ctx.matchesTags` |

## 3. 迁移是怎么验的(留个记录)

**金标准 harness(已退役删除)**:v5.2 刚废除的东西请回来打了一仗。

- 钉死 `testDate`,对 **11 条代表性 URL**(单域/混册/tags/exact/allday/merge 开关/json)
  抓 v5 语义快照;
- **比语义不比字节**:折行、属性次序、转义形式、DTSTAMP、uid 前缀 —— 全部归一化抹平;
  JSON 侧引用 `governance.js` 同一个 `checkAlarmUid` 双侧剔除(单一真相源,不搞两套规则)。
- **手术 A(换交付形态,行为须全等)验收:11/11 语义全等。**
- **手术 B(有意改行为)前退役删除** —— 服役期到此为止,再比就是自缚。退役前最后一次 diff:
  **属性级差异 0 条**,全部差异都是 uid 换编号,正是手术 B 的目的。

**测试**:v5 基线 60 绿 → v6 **185 项**(框架层 72 + 域层 113),零回归。

永久兼容层让人忘记欠债,那座桥让欠债持续刺眼。card 手术 A 完成即拆,现已归零
(worker-entry 分支 + render.test 的 MIGRATION_WHITELIST 同时清空,两者互为进度条)。

## 4. 回滚

`git revert` 到 v5.2 即可 —— 但**注意 KV**:若你在 v6 上线后启用了 email 模型并写入了数据,
回滚会让 v5 按旧键找不到记录(表现为"等待首封账单邮件")。
在启用 email 模型**之前**回滚,无任何副作用。
